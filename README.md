# Storage Manager

A simple Go-based utility for managing local storage and identifying unnecessary files for cleanup.

## Features

- **Size Breakdown**: Get a detailed breakdown of directory and file sizes within a specified folder.
- **Cleanup Identification**: Scan directories for common temporary and unnecessary files (e.g., `venv`, `node_modules`, `__pycache__`, `.DS_Store`, etc.).
- **Automated Cleanup**: Remove identified files to free up space.
- **Docker Integration**: Check Docker storage usage across different platforms.

## Installation

Ensure you have [Go](https://golang.org/doc/install) installed (version 1.25.3 or later recommended).

### CLI Tool

You can install the CLI tool directly using `go install`:

```bash
go install github.com/Samar2170/storage-manager/cmd/storage-manager@latest
```

Alternatively, clone the repository and build it:

1. Clone the repository:
   ```bash
   git clone https://github.com/Samar2170/storage-manager.git
   cd storage-manager
   ```

2. Build the tool:
   ```bash
   go build -o storage-manager ./cmd/storage-manager
   ```

### As a Package

To use `storage-manager` as a library in your Go project:

```bash
go get github.com/Samar2170/storage-manager
```

## Usage

### CLI Usage

The tool provides several commands for managing storage:

#### 1. Get Directory Size Breakdown

Display the sizes of all subdirectories and files within a given folder.

```bash
storage-manager size-info -F /path/to/folder
```

#### 2. Check for Cleanup Opportunities

Scan a folder for unnecessary files.

```bash
storage-manager check-cleanup -F /path/to/folder
```

#### 3. Perform Cleanup

Execute the cleanup based on the temporary files generated by the `check-cleanup` command.

```bash
storage-manager cleanup
```

#### 4. Organize Photos

Organize photos into folders based on location, camera, and orientation.

```bash
storage-manager organize-photos -S /source/path -D /dest/path
```

## Cleanup Patterns

The tool identifies the following patterns for cleanup:

- **Environments**: `venv`, `*venv`, `*env`, `node_modules`
- **Caches & Metadata**: `__pycache__`, `.git`, `.idea`, `.vscode`, `.DS_Store`
- **Build Artifacts**: `dist`, `build`, `*.egg-info`, `*.dist-info`
- **Temporary Files**: `*.log`, `*.tmp`, `*.bak`, `*.swp`, `*.swo`
- **Package Files**: `*.tar.gz`, `*.whl`, `*.dmg`, `*.iso`, `*.exe`, `*.msi`, `*.deb`, `*.rpm`, `*.apk`, `*.ipa`
